RESPUESTA DE INTEGRACIÓN DE API EXTERNA (RESUMIDA)

1) Selección de API externa
Se integró una API REST de tipo de cambio llamada Frankfurter (https://api.frankfurter.app). Esta API permite convertir monedas (por ejemplo, USD a MXN) y no requiere configuración compleja para pruebas básicas.

Se eligió porque es útil para el proyecto: permite convertir precios de referencia y mostrar valores en otra moneda de forma simple, lo cual puede ayudar en decisiones de inventario y costos.

2) Ejemplo de IA y adaptación en Node.js
Se consultó a una IA un ejemplo básico para consumir API externa en Node.js con Express. La idea principal fue: hacer la petición con fetch, validar respuesta, transformar datos útiles y manejar errores cuando la API externa falle.

Ese ejemplo se adaptó a nuestro backend en rutas reales y protegidas por JWT, agregando validación de parámetros y control de errores. Así se aplicaron buenas prácticas: manejo de errores (try/catch y errores HTTP), validación de datos y respuesta clara para frontend.

3) Funcionalidad creada con datos de API externa
Se crearon dos funcionalidades: una ruta para conversión general de monedas (`/externa/convertir`) y otra enfocada al proyecto para convertir precio de producto de USD a MXN (`/externa/precio-producto-mxn`).

Con esto, la aplicación ya usa datos reales de una API externa y los procesa para una necesidad práctica. Además, estas rutas están protegidas con JWT y validadas, por lo que no solo funcionan, también son más seguras y mantenibles.
